  • Form properties and methods

    • submit() - Method that submits the form automatically.
    • reset() - Method that resets all form controls to their initial values specified in the HTML.
    • action - Property that specifies the URL to submit the form to.
    • method - Property that specifies the HTTP method to use when submitting the form. If GET is used, the form values will be URL parameters.
    • focus - Event that occurs when an element is focused on.
    • blur - Event that occurs when an element loses focus.
    • changed - Event that occurs when an element loses focus and its value has been changed.
    • submit - Event that occurs when a form is submitted.

Chapter 12: Object-Oriented Programming in JavaScript

  • Three main concepts

    • Encapsulation - Keep the programming logic inside an object nad make methods available to implement the functionality, without the outside world needing to know how it's done.
    • Polymorphism - Various objects share the same method, but also have the ability to override shared methods with a more specific implementation.
    • Inheritance - Take an object that already exists and inherit all its properties and methods.

  • Constructor function sample

    const dice = { sides: 6, roll() { return Math.floor(this.sides * Math.random() + 1) } }

  • Class sample

    class Turtle { constructor(name) { this.name = name; this.weapon = 'hands'; } sayHi() { return `Hi dude, my name is ${this.name}`; } attack(){ return `Feel the power of my ${this.weapon}!`; } }

  • Accessing prototype

    • raph.constructor.prototype
    • Object.getPrototypeOf(raph)
    • raph.__proto__
  • The prototype object is live, so if a new property or method is added to the prototype, any instances of its class will inherit the new properties and methods automatically.
  • An object instance can overwrite any properties or methods inherited from its prototype by simply assigning a new value to them.
  • Objects can be inherited by using the extends keyword.

  • Property attributes an descriptors

    • value - The value of the property and is undefined by default.
    • writable - This boolean value shows whether a property can be changed or not, and is false by default.
    • enumerable - This boolean value shows whether a property will show up when th eobject is displayed in a for in loop, and is false by default.
    • configurable - This boolean value shows whether you can delete a property or change any of its attributes, and is false by default.

  • Property methods

    • propertyIsEnumerable() - Return true for user-created properties and methods.
    • hasOwnProperty() - Return true for properties that belong directly to an object.
    • getOwnPropertyDescriptor() - Return an object of property descriptors.
    • Object.defineProperty() - Define a new property along with property descriptor values.
  • Object.create() - Create object using other object as prototype.

  • Mixins

    • Mixin - A way of adding properties and methods of some object to another object without using inheritance.
    • Object.assign() - Method that provides basic mixin functionality. Shallow copy only.
