Week 5

Chapter 10: Testing and Debugging

  • What are the types of errors in a program?

    • System error - there’s a problem with the system or external devices with which the program is interacting.
    • Programmer error ― the program contains incorrect syntax or faulty logic; it could even be as simple as a typo.
    • User error ― the user has entered data incorrectly, which the program is unable to handle.

  • How can I use a stack trace?

    After pausing the code execution at a breakpoint, I can then navigate through the hierarchy of functions and methods that were called leading up to that point.

  • What is a warning?

    A warning indicates a possible problem that is not severe enough to stop the program from running.

  • How can I debug in a web browser?

    • alert() - Use the alert function to pause execution and display values.
    • console.log() - Print text to the console.
    • console.trace() - Print a stack trace to the console.
    • debugger keyword - debugger will cause the code to break when it is encountered.

  • Error Objects

    • EvalError - Deprecated. Used to identify errors in the eval() function.
    • RangeError - Thrown when a number is outside an allowable range of values.
    • ReferenceError - Thrown when a reference is made to an item that does not exist.
    • SyntaxError - Thrown when there's an error in the code's syntax.
    • TypeError - Thrown when there's an error in the type of value used.
    • URIError - Thrown when there's a problem encoding or deocidng the URI.
    • InternalError - Non-standard error that is thrown when an error occurs in the JavaScript engine.

  • Common error object properties (Safe across browsers)

    • name - Returns the name of the error contructor function.
    • message - Returns a description of the error and should be provided as an argument to the Error constructor function.
    • stack - Return a stack trace for the error.
